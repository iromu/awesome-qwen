# DSL Builders

Embabel agents can be created via a type-safe DSL in **Java** or **Kotlin**. This is useful for atomic, self-contained workflows that may contain multiple steps or actions.

Builders are used to create agents that can be exposed as Spring beans or invoked within `@Action` methods of other agents. All builders share a consistent API.

---

## SimpleAgentBuilder

The simplest agent — no preconditions or postconditions.

```java
// Java
var agent = SimpleAgentBuilder
    .returning(Joke.class)
    .running(tac -> tac.ai()
        .withDefaultLlm()
        .createObject("Tell me a joke", Joke.class))
    .buildAgent("joker", "Returns a dreadful joke");
```

```kotlin
// Kotlin
val agent = SimpleAgentBuilder
    .returning(Joke::class.java)
    .running { tac -> tac.ai()
        .withDefaultLlm()
        .createObject("Tell me a joke", Joke::class.java) }
    .buildAgent("joker", "Returns a dreadful joke")
```

| Step | Purpose |
|------|---------|
| `.returning(T.class)` | Specify the return type |
| `.running(SupplierActionContext<RESULT> op)` | Define the action; receives `OperationContext` with access to `AgentProcess` |
| `.buildAgent(name, description)` | Build the `Agent` bean |

## ScatterGatherBuilder

Fork-join pattern for parallel processing over a collection.

```java
// Java — within an @Action method
FactChecks runAndConsolidateFactChecks(
        DistinctFactualAssertions assertions,
        ActionContext context) {
    var llmFactChecks = properties.models().stream()
            .flatMap(m -> factCheckWithSingleLlm(m, assertions, context))
            .toList();
    return ScatterGatherBuilder
            .returning(FactChecks.class)
            .fromElements(FactCheck.class)
            .generatedBy(llmFactChecks)
            .consolidatedBy(this::reconcileFactChecks)
            .asSubProcess(context);
}
```

```kotlin
// Kotlin
@Action
fun runAndConsolidateFactChecks(
    assertions: DistinctFactualAssertions,
    context: ActionContext
): FactChecks {
    val llmFactChecks = properties.models
        .flatMap { model -> factCheckWithSingleLlm(model, assertions, context) }
    return ScatterGatherBuilder
        .returning(FactChecks::class.java)
        .fromElements(FactCheck::class.java)
        .generatedBy(llmFactChecks)
        .consolidatedBy(this::reconcileFactChecks)
        .asSubProcess(context)
}
```

| Step | Purpose |
|------|---------|
| `.returning(T.class)` | Overall return type |
| `.fromElements(E.class)` | Element type generated by each fork |
| `.generatedBy(List<Supplier<E>>)` | Parallel fork functions |
| `.consolidatedBy(Function<List<E>, T>)` | Merge all results |
| `.asSubProcess(context)` | Run inline (alternative to `.buildAgent()`) |

> **Example repo:** [embabel-agent-examples](https://github.com/embabel/embabel-agent-examples) — [FactChecker.java](https://github.com/embabel/embabel-agent-examples/blob/main/examples-java/src/main/java/com/embabel/example/factchecker/FactChecker.java)

## ConsensusBuilder

Reaches consensus among multiple sources. A specialization of `ScatterGatherBuilder`.

```java
var agent = ConsensusBuilder
    .returning(ConsensusResult.class)
    .fromElements(Opinion.class)
    .generatedBy(opinionSources)
    .consolidatedBy(this::reachConsensus)
    .buildAgent("consensus", "Reach consensus from multiple opinions");
```

---

## RepeatUntil

Repeats a step until a condition is met.

```java
var agent = RepeatUntil
    .returning(Draft.class)
    .withStep(ctx -> generateDraft(ctx.getInput()))
    .withCondition(draft -> draft.isAcceptable())
    .buildAgent("draft-until-ok", "Iterate until draft is acceptable");
```

---

## RepeatUntilAcceptable

Repeatedly executes a step while a condition holds, using a **separate evaluator** to provide feedback.

```java
var agent = RepeatUntilAcceptable
    .returning(Refined.class)
    .withStep(ctx -> refineDraft(ctx.getInput()))
    .withEvaluator(result -> evaluateQuality(result))
    .buildAgent("refine", "Refine until quality threshold met");
```

## Registering Agent Beans

The `@Agent` annotation auto-registers a class as a Spring bean. With the DSL, register agents via a `@Bean` method that returns `Agent`:

```java
// Java
@Configuration
public class FactCheckerAgentConfiguration {

    @Bean
    public Agent factChecker(FactCheckerProperties props) {
        return factCheckerAgent(
            List.of(
                LlmOptions.create(AnthropicModels.CLAUDE_35_HAIKU).withTemperature(0.3),
                LlmOptions.create(AnthropicModels.CLAUDE_35_HAIKU).withTemperature(0.0)
            ),
            props
        );
    }
}
```

```kotlin
// Kotlin
@Configuration
class FactCheckerAgentConfiguration {

    @Bean
    fun factChecker(factCheckerProperties: FactCheckerProperties): Agent {
        return factCheckerAgent(
            llms = listOf(
                LlmOptions(AnthropicModels.CLAUDE_35_HAIKU).withTemperature(0.3),
                LlmOptions(AnthropicModels.CLAUDE_35_HAIKU).withTemperature(0.0),
            ),
            properties = factCheckerProperties,
        )
    }
}
```

Any `@Bean` method returning `Agent` triggers auto-registration, equivalent to a class annotated with `@Agent`.

## Quick Reference

| Builder | Use case |
|---------|----------|
| `SimpleAgentBuilder` | Single-step agent, no pre/post conditions |
| `ScatterGatherBuilder` | Parallel fork-join over a collection |
| `ConsensusBuilder` | Reach consensus from multiple sources |
| `RepeatUntil` | Loop until a condition becomes true |
| `RepeatUntilAcceptable` | Loop with separate evaluator/feedback |

All builders are type-safe and share the `.returning()`, `.buildAgent(name, description)` API. Use `.asSubProcess(context)` to run inline within another agent's execution.
